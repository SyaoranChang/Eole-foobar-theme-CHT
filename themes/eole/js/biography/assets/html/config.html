文字閱讀器和歌詞說明
一般
  • 使用必須被啟用（左上角）
  • 圖片和其他資源都是可用的，所以文字和歌詞可以用更有趣的方式來查看
  • 檢查"歌詞來源"以確定哪些來源需要處理歌詞
歌詞
- 自動捲動現有的和新下載的歌詞 - 默認情況是，如果沒有找到歌詞，則退回到顯示傳記或評論。 - 支援歌詞偏移+用滑鼠滾輪飛行調整（不保存）。
下載整合
傳記不下載歌詞，但可以與歌詞外掛同步，如下所示。
  • 勾選上面的 "同步..."。
  • 確保在佈局中設定了 lyric show 3 或公開指標面板（可以隱藏），並啟用了歌詞保存功能。
  • 確保歌詞面板中的儲存設定與上面的載入設定一致
如果使用 lyric show 3，請安裝 veksha 的 multi-source 來使用歌詞。停用默認的3個歌詞來源，因為它們大多不再起作用了。啟用NEW DB: Minilyrics, NEW DB: QQ Music & NEW DB: Netease來同步歌詞或根據需要啟用其他來源。
淡出的高度
如果淡化效果太接近頂部/底部，則增加，例如，嘗試0.5。

  • 對於 %lyrics% 或 %unsynced lyrics% 標籤，請註釋或刪除 "fieldSpam"下的這些特定條目。

標記器的說明
一般
所有標籤的編寫都來自保存的資料（即只編寫已經保存的簡介和評論中的現有資訊）。這是為了遵守某些網站的限制。如果傳記已經被使用了一段時間，並且項目被快取了足夠長的時間，那麼標籤器應該工作得最好。標籤是多值的，除了評分。
Last.fm 音樂類型
白名單用於刪除任何無音樂類型的標籤。翻譯也是可用的。其他音樂類型的資源不需要這些。
統計資料
標記統計需要用傳記v1.2.0或更高版本建立或更新的檔案。 Last.fm 的播放次數（scrobbles）、聽眾和綜合得分（1-100）被寫成一個多值標籤。
分數將藝術家和專輯的長期受歡迎程度在1-100的範圍內用總的瀏覽量和聽眾進行排名。在 last.fm 得分最高的藝術家或專輯被設定為100分。

通知標籤
如果選中，當前曲目的啟用標籤將被通知給其他 Spider Monkey Panels 的面板。這包括廣播流的藝術家資料。當收到新的或更新的資料時，會在曲目變化時傳送通知。 根據面板的顯示模式，當前的軌道將是正在播放的軌道或選定的軌道。如果有一個以上的傳記面板，不同的面板可能有所不同。選擇模式和控制代碼被傳送。使用默認的標籤名稱，而不是任何自訂的名稱，所以第三方指令碼知道該怎麼做。 在接收面板中使用 on_notify_data(name, info) 來處理通知，在: name is 'biographyTags' info is an object: { handle: handle of current track, selectionMode: biography selectionMode - string either 'Prefer nowplaying' or 'Follow selected track (playlist)' tags: { {name: value}, {name: value}... } } 第三方指令碼可以通過使用 window.NotifyOthers('bio_followSelectedTrack', boolean) 來設定傳記選擇模式*，其中boolean是true[跟隨選定的曲目（播放列表）]或false[傾向於現在播放]。*如果啟用了歌詞來源，則不適用，因為面板必須處於偏好現在播放的模式。
